# 更新日志 (v7.1.0) - 文件上传与多媒体增强

## 核心更新
本次更新解决了 Tiptap 编辑器在处理大文件时的性能瓶颈，并增强了多媒体交互体验。

### 1. 突破限制：分片上传系统
- **解决 413 错误**：通过将文件切分为 **512KB** 的数据块进行异步上传，彻底绕过了 Nginx 及后端对请求体大小的限制。
- **断点合片**：后端新增 `init/chunk/complete` 接口，支持大文件的分块接收与自动合并，确保上传稳定性。
- **本地存储**：上传后的文件存储于服务器 `data/uploads` 目录，笔记仅保存轻量级 URL 引用。

### 2. 秒开体验：Optimistic UI 预览
- **即时预览**：拖入文件后利用 `Blob URL` 瞬间生成本地预览（图片/视频/音频），无需等待上传完成即可继续编辑。
- **自动同步**：后台上传完成后，编辑器会自动定位对应的占位符并丝滑替换为正式服务器 URL，并提供透明度渐变反馈。

### 3. 多媒体节点升级
- **全量 Node 化**：图片、视频、音频均重构为 Tiptap 自定义 Node，支持 `data-upload-id` 跟踪与状态同步。
- **安全清洗**：新增保存拦截逻辑，自动清除笔记中可能存在的失效 `blob:` 链接，确保数据库数据纯净、可迁移。
- **文件卡片 (File Node)**：为非媒体文件（PDF、Zip 等）或超大文件提供 Notion 风格的精致文件卡片，支持一键下载与类型识别。

### 4. 智能链接识别 (Embed)
- **粘贴预览**：自动识别剪贴板中的 **Bilibili** 和 **YouTube** 链接。
- **自动转播放器**：识别成功后自动将其转换为嵌入式播放器（Iframe），实现“所见即所得”的在线视频笔记体验。

## 修复与优化
- 修复了 `RangeError: Content hole not allowed` 导致的编辑器崩溃问题。
- 修复了 `blob: ERR_FILE_NOT_FOUND` 导致的预览图消失问题。
- 优化了前端构建流程，确保最新的分片逻辑覆盖全量产物。
